# AP-017: rules 블록 반복

| 항목 | 값 |
|------|-----|
| 심각도 | info |
| 카테고리 | 구조 |
| 자동 수정 | — |
| 도입 | v0.1.0 |

## 설명

동일한 `rules` 블록이 3개 이상의 job에서 반복됩니다. hidden job이나 `!reference`로 추출하면 유지보수가 쉬워집니다.

## 근거

`rules`는 가장 자주 복사-붙여넣기되는 블록입니다. 조건을 변경할 때 모든 job을 수정해야 하고, 하나를 빠뜨리면 파이프라인 동작이 달라집니다.

## 탐지

`rules` 배열의 구조적 fingerprint(if, changes, exists, when, allow_failure)를 비교합니다. 3개 이상의 job이 동일한 fingerprint를 가지면 경고합니다.

## 예시

```yaml
# Bad — 동일한 rules가 3곳에 복사됨
build:
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"

test:
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"

lint:
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"

# Good — hidden job으로 추출
.default-rules:
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"

build:
  rules: !reference [.default-rules, rules]

test:
  rules: !reference [.default-rules, rules]

lint:
  rules: !reference [.default-rules, rules]
```
