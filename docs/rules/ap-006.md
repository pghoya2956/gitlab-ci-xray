# AP-006: 과도한 artifacts 범위

| 항목 | 값 |
|------|-----|
| 심각도 | warning |
| 카테고리 | 스토리지 |
| 자동 수정 | — |
| 도입 | v0.1.0 |

## 설명

`artifacts.paths`가 `**/*`, `*`, `build/` 같은 넓은 패턴을 사용하면 불필요한 파일이 artifacts에 포함되고, downstream job이 `dependencies`를 지정하지 않으면 모든 파일이 자동 다운로드됩니다.

## 근거

넓은 artifacts는 두 가지 비용을 유발합니다:
- **스토리지**: 불필요한 파일이 저장됨
- **네트워크**: 다음 stage의 모든 job이 전체 artifacts를 다운로드

> When you don't specify a `dependencies` keyword in a job, all artifacts from all previous stages are passed.
> — [GitLab CI dependencies](https://docs.gitlab.com/ee/ci/yaml/#dependencies)

## 예시

```yaml
# Bad
build:
  artifacts:
    paths:
      - "**/*"

# Good
build:
  artifacts:
    paths:
      - dist/bundle.js
      - dist/index.html
    expire_in: 1 day
```
