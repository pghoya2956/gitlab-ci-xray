# AP-008: resource_group 미설정

| 항목 | 값 |
|------|-----|
| 심각도 | info |
| 카테고리 | 안전성 |
| 자동 수정 | — |
| 도입 | v0.1.0 |

## 설명

`environment`를 사용하는 deploy job에 `resource_group`이 없으면 동시 배포가 발생할 수 있습니다.

## 근거

같은 환경에 여러 파이프라인이 동시에 배포하면 예측 불가능한 상태가 됩니다. `resource_group`을 설정하면 동일 리소스에 대해 한 번에 하나의 job만 실행됩니다.

> Use `resource_group` to create a mutual exclusion lock to ensure only one job in a resource group runs at a time.
> — [GitLab resource_group](https://docs.gitlab.com/ee/ci/yaml/#resource_group)

## 예시

```yaml
# Bad
deploy:
  environment: production
  script: ./deploy.sh

# Good
deploy:
  environment: production
  resource_group: production
  script: ./deploy.sh
```
