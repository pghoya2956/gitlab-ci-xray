# AP-001: only/except 사용

| 항목 | 값 |
|------|-----|
| 심각도 | warning |
| 카테고리 | 구조 |
| 자동 수정 | — |
| 도입 | v0.1.0 |

## 설명

`only`/`except` 키워드 대신 `rules` 사용을 권장합니다.

## 근거

GitLab 공식 문서에서 `only`/`except`를 **deprecated** 로 표기하고 있습니다. `rules`가 더 유연하고 예측 가능한 동작을 제공합니다.

> `only` and `except` are not being actively developed. `rules` is the preferred keyword to control when to add jobs to pipelines.
> — [GitLab CI/CD YAML reference](https://docs.gitlab.com/ee/ci/yaml/#only--except)

## 차이점

| only/except | rules |
|-------------|-------|
| 조건 조합이 AND/OR 혼합되어 예측 어려움 | 순차 평가, 첫 매칭 규칙 적용 |
| `changes`와 `refs` 조합 시 버그 다발 | `if`, `changes`, `exists` 명확한 조합 |
| 신규 기능 추가 없음 | `needs`, `variables` 등 지속 확장 |

## 예시

```yaml
# Bad
build:
  only:
    - main
    - merge_requests
  script: npm run build

# Good
build:
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
  script: npm run build
```
