# AP-007: retry 미설정 (네트워크 의존)

| 항목 | 값 |
|------|-----|
| 심각도 | warning |
| 카테고리 | 안정성 |
| 자동 수정 | `retry: { max: 2, when: runner_system_failure }` 추가 |
| 도입 | v0.1.0 |

## 설명

script에 네트워크 접근 명령(`npm ci`, `curl`, `apt install`, `docker pull` 등)이 있지만 `retry`가 설정되지 않아 일시적 네트워크 오류에 취약합니다.

## 근거

네트워크 의존 명령은 DNS 실패, 레지스트리 일시 장애, 미러 불안정 등으로 간헐적 실패가 발생합니다. `retry`를 설정하면 flaky pipeline을 크게 줄일 수 있습니다.

> Use `retry` to configure how many times a job is retried in case of failure.
> — [GitLab CI retry](https://docs.gitlab.com/ee/ci/yaml/#retry)

## 탐지 패턴

다음 패턴이 script에 있으면 네트워크 의존으로 판단합니다:

`curl`, `wget`, `apt install`, `yum install`, `dnf install`, `apk add`, `npm install`, `npm ci`, `pip install`, `gem install`, `go get`, `go mod download`, `docker pull`, `helm repo/pull/install/upgrade`

## 예시

```yaml
# Bad
install:
  script:
    - npm ci

# Good
install:
  script:
    - npm ci
  retry:
    max: 2
    when: runner_system_failure
```
