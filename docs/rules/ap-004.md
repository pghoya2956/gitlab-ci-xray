# AP-004: cache key 미설정

| 항목 | 값 |
|------|-----|
| 심각도 | warning |
| 카테고리 | 캐시 |
| 자동 수정 | 파일 기반 key 추가 |
| 도입 | v0.1.0 |

## 설명

`cache`에 `key`가 설정되지 않으면 모든 job이 `default` 키로 같은 캐시를 공유하여 충돌이 발생할 수 있습니다.

## 근거

> If not set, the default key is `default`. All jobs that use the same cache key use the same cache, including in different pipelines.
> — [GitLab CI cache](https://docs.gitlab.com/ee/ci/caching/#cache-key)

lockfile 기반 `key: files:` 를 사용하면 의존성이 변경될 때만 캐시가 갱신됩니다.

## 예시

```yaml
# Bad
build:
  cache:
    paths:
      - node_modules/

# Good
build:
  cache:
    key:
      files:
        - package-lock.json
    paths:
      - node_modules/
```
