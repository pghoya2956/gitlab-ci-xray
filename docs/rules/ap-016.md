# AP-016: 파괴적 명령 + 미검증 변수

| 항목 | 값 |
|------|-----|
| 심각도 | warning |
| 카테고리 | 안전성 |
| 자동 수정 | — |
| 도입 | v0.1.0 |

## 설명

`rm -rf`, `rsync --delete`, `mv`, `cp -r` 등 파괴적 명령이 환경 변수를 참조하지만, 변수가 비어있을 때의 방어 코드가 없습니다.

## 근거

변수가 빈 값이면:
- `rm -rf ${DEPLOY_DIR}/*` → `rm -rf /*` (루트 삭제)
- `rsync --delete ./ ${REMOTE}/` → `rsync --delete ./ /` (루트 동기화)

이는 CI/CD 환경에서 실제로 발생하는 사고 패턴입니다.

## 탐지 패턴

다음 명령이 `$VAR` 형태의 변수와 함께 사용될 때 탐지합니다:

`rm -rf`, `rm -f`, `rsync --delete`, `mv`, `cp -r`, `chmod -R`, `chown -R`

## 안전 패턴

다음 중 하나라도 script에 있으면 경고하지 않습니다:

- `${VAR:?error message}` — 변수가 비어있으면 에러로 중단
- `${VAR:-default}` — 기본값 제공
- `[ -n "$VAR" ]` — 빈 값 체크
- `[[ -n "$VAR" ]]` — bash 빈 값 체크
- `test -n "$VAR"` — POSIX 빈 값 체크

## 예시

```yaml
# Bad
deploy:
  script:
    - rm -rf ${DEPLOY_DIR}/*

# Good — 변수 검증
deploy:
  script:
    - rm -rf ${DEPLOY_DIR:?DEPLOY_DIR must be set}/*

# Good — 조건 체크
deploy:
  script:
    - '[ -n "$DEPLOY_DIR" ] || exit 1'
    - rm -rf ${DEPLOY_DIR}/*
```
