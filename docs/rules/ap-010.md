# AP-010: interruptible 미설정

| 항목 | 값 |
|------|-----|
| 심각도 | info |
| 카테고리 | 성능 |
| 자동 수정 | — |
| 도입 | v0.1.0 |

## 설명

`interruptible: true`를 설정하면 새 커밋 push 시 이전 파이프라인의 해당 job이 자동 취소되어 러너 자원을 절약합니다.

## 근거

개발 중 빠르게 커밋을 push하면 이전 파이프라인이 여전히 실행 중일 수 있습니다. `interruptible`을 설정하면 낭비되는 러너 시간을 줄입니다.

## 예외

- `when: manual` job은 제외 (사용자 의도적 실행)
- `environment` 있는 job은 제외 (배포 중 취소 위험)
- `default.interruptible`이 설정되어 있으면 전체 제외
