# AP-011: script 중복

## 설명

여러 job에 유사한 script(70% 이상 일치)가 있습니다. `extends`나 `!reference`로 공통 부분을 추출하면 유지보수가 쉬워집니다.

## 근거

DRY(Don't Repeat Yourself) 원칙. 동일한 script를 여러 job에 복사하면 변경 시 모든 곳을 수정해야 하고, 누락되면 동작이 달라집니다.

## 예시

```yaml
# Bad
build-a:
  script:
    - npm ci
    - npm run build
    - npm run test

build-b:
  script:
    - npm ci
    - npm run build
    - npm run lint

# Good
.common:
  before_script:
    - npm ci
    - npm run build

build-a:
  extends: .common
  script: npm run test

build-b:
  extends: .common
  script: npm run lint
```
