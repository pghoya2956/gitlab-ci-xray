# AP-015: allow_failure + rules 혼용

| 항목 | 값 |
|------|-----|
| 심각도 | info |
| 카테고리 | 가독성 |
| 자동 수정 | — |
| 도입 | v0.1.0 |

## 설명

job 레벨의 `allow_failure`와 `rules[].allow_failure`가 동시에 존재하면 어떤 값이 적용되는지 혼란스러울 수 있습니다.

## 근거

`rules`에 `allow_failure`가 지정되면 해당 rule이 매칭될 때 **rules의 값이 우선**합니다. job 레벨 `allow_failure`는 rules에 `allow_failure`가 없는 rule이 매칭될 때만 적용됩니다. 이 우선순위가 직관적이지 않아 실수를 유발합니다.
