# AP-002: artifacts expire_in 미설정

| 항목 | 값 |
|------|-----|
| 심각도 | warning |
| 카테고리 | 스토리지 |
| 자동 수정 | `expire_in: 1 week` 추가 |
| 도입 | v0.1.0 |

## 설명

`artifacts`에 `expire_in`이 설정되지 않으면 기본 30일간 보관되어 스토리지를 낭비합니다.

## 근거

GitLab 인스턴스의 스토리지는 유한합니다. 특히 Self-managed GitLab에서 artifacts가 누적되면 디스크 부족 문제가 발생합니다. SaaS(gitlab.com)에서도 스토리지 할당량에 영향을 줍니다.

> The default expiry time is 30 days. To set a shorter expiry, use `expire_in`.
> — [GitLab CI/CD artifacts](https://docs.gitlab.com/ee/ci/jobs/job_artifacts.html)

## 예외

- `artifacts.reports` (JUnit, Cobertura 등)는 GitLab UI에서 소비되므로 `expire_in` 없어도 경고하지 않습니다.

## 예시

```yaml
# Bad
build:
  artifacts:
    paths:
      - dist/

# Good
build:
  artifacts:
    paths:
      - dist/
    expire_in: 1 week
```
