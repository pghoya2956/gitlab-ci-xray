# AP-003: 병렬화 가능한 job

| 항목 | 값 |
|------|-----|
| 심각도 | info |
| 카테고리 | 성능 |
| 자동 수정 | — |
| 도입 | v0.1.0 |

## 설명

같은 stage에 있지만 서로 의존하지 않는 job들은 `needs`를 사용하면 이전 stage 전체 완료를 기다리지 않고 더 빨리 시작할 수 있습니다.

## 근거

GitLab CI의 기본 동작은 이전 stage의 **모든** job이 완료된 후 다음 stage를 시작합니다. `needs`로 명시적 의존성을 지정하면 필요한 job만 기다리고 바로 실행합니다 (DAG 모드).

> Use `needs` to execute jobs out of order. Relationships between jobs that use `needs` form a directed acyclic graph (DAG).
> — [GitLab DAG](https://docs.gitlab.com/ee/ci/directed_acyclic_graph/)

## 예시

```yaml
# Before: test-unit은 build stage 전체 완료를 기다림
build-frontend:
  stage: build
  script: npm run build

build-backend:
  stage: build
  script: mvn package

test-unit:
  stage: test
  script: npm test  # build-backend 완료도 기다림 (불필요)

# After: test-unit은 build-frontend만 기다림
test-unit:
  stage: test
  needs: [build-frontend]
  script: npm test
```
